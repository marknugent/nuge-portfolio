Cisco Cross-Domain Automation

No-code SaaS workflow tool for networking applications

Desktop web app

My role: conceptual design, wireframes, interaction design

Background

Cisco is creating a new product for automating processes across networking domains. The new product, currently called "Cross-Domain Automation," will allow users to create workflows that automate tasks across disparate networking domains without using code. It’s sort of an IFTTT for networking.

 

My Role

I serve as a senior interaction designer on a design team of interaction and visual designers. We worked closely with product managers during our design process.

In addition to core interaction design work, I often create conceptual diagrams, taxonomies, and other deliverables in order to drive consensus regarding users, goals, and in-product concepts.

 

My Work

Earlier in the project I worked on the conceptual side, working with our project managers to refine CDA’s requirements and help define the product conceptually.

In terms of UI design, my role is to create screens and UI flows for one of the primary personas: the network operator (whom we call Tarek). This persona’s tasks include viewing, running, and monitoring workflows. Shown here are examples of UI designed to facilitate those tasks.

Using Figma, I worked within Cisco’s Magnetic design system to design screens and create a prototype. The prototype was sufficiently rich to allow testers to explore it freely in an unmoderated user test.
