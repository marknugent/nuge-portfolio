Background

The market for CPU cores is dominated by Intel in traditional PCs and servers, and ARM in smartphones and specialized applications such as embedded processors. SiFive, a startup semiconductor company, challenges the Intel-ARM duopoly with the open-source RISC-V instruction set architecture.

SiFive's founders invented RISC-V as a research project at Berkeley. At the same time, they also developed new methods for automating the configuration and generation of CPU core designs. This new technology enables the creation of unique cloud-based products for core IP and microchip design.

When I began work at SiFive as their first (and so far, only) product designer, I was put to work on their planned SaaS products: Core Designer, for configuring CPU cores (the part of a microchip that does general computing tasks), and Chip Designer, for configuring and designing finished microchips, which normally integrate one or more CPU cores as well as various other functional blocks.

Designing Core Designer

After extensive discussions with CPU design engineers, prospective users, and other stakeholders, I created a set of proto-personas representing our target users, and then began work on UI concepts. Because it is a farily unique product, I needed to solve some interesting problems due to constraints relating to CPU design generally and our technology in particular.

The general pattern is that users choose a standard design as a starting point, which is populated with default values, and then add values and adjust settings as they desire. They can then generate and download a development kit for their core which contains a representation of the core in code that canÂ can be integrated into various simulation and development tools.

With Core Designer, we aim to allow users to quickly iterate and experiment with CPU core designs in a way that is not possible in the current industry model, where customers might be limited to a vendor’s off-the-shelf cores, or at best, work with salespeople to get customized cores prepared by engineers — a time-consuming and expensive process.

Initially, I sketched UI ideas on paper and used Balsamiq to create wireframes. I settled upon using a tabbed interface for the main UI, with related settings grouped together in tabs.

When consensus had formed around basic UI patterns, I wanted to create a mockup with enough fidelity to be plausible as a real UI, so I created a set of medium-fidelity mockups using Sketch. I also created a number of clickable prototypes using InVision and later Figma, using these to gather feedback and conduct hallway usability testing.

Visual Design and Launch

Before the launch of Core Designer, SiFive had engaged with a design firm to create SiFive's new brand style and website. In consultation with the design firm, the visual design of Core Designer was adjusted to adopt the website's visual style. For example, the "start page" shown here uses signature graphics that the design firm had created to give each of our CPU cores a unique graphic identity.

Core Designer launched in late 2018 and is used internally at SiFive and at companies such as Intel and Qualcomm. Not only a revenue-generating product in its own right, Core Designer is also employed in our sales process, allowing prospective customers to quickly spec out and generate CPU cores for evaluation.

Iteration and Improvement

Following the launch, we have steadily added features and enhanced the Core Designer UI. I also began using Figma as a design and collaboration tool. Pictured here is a detail from a set of mockups I prepared to advocate for the addition of a range slider to the UI, and also to describe its behavior for the developer who subsequently implemented it.

I also have acted as a member of the development team, having learned enough about our codebase, Django, React, and Styled Components to implement some UI enhancements and fixes without involving other developers, save for the pull-request stage.

Chip Designer

Chip Designer is an ambitious product, not yet launched, that will allow its users to create customized microchips through a cloud-based application.

As with Core Designer, I began with sketches on paper and then created wireframes in Balsamiq in collaborative process.

I designed the main UI (shown in the image with the "El Camino Real" header) to follow a similar pattern of applications such as XCode and Keynote, with a main canvas in the center showing a chip’s block diagram as the "document," and a properties panel on the right. However, Chip Designer differs from those applications in that a left panel is used to show a library of IP blocks that are available to include on the chip, whereas apps like XCode and Keynote use this area for navigation through the document or project.

The properties area initially shows general information about the chip, but when a particular IP block on the chip is selected, the properties area updates with settings and properties for the selected IP.

The "Area Inspector" shown in the final image is a concept for a little utility, revealed in a modal pop-up, that gives a detailed view of how much space various IP blocks are taking up on a chip. Managing the area budget is one of the more important aspects of SoC (system-on-chip) design. The pattern is borrowed from operating system utilities for managing hard drive space.
